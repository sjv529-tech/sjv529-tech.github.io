📝 Development Diary

a reflective log of my web development journey (weeks 1-16)

ella's coding journey

this diary documents my progress throughout the IY4103 Web Development module. entries are written weekly to reflect on challenges, achievements, and learning outcomes.

week 1 — 6th january 2026
🌱 introduction to web development
html basics setup

This week marked the beginning of the module. We were introduced to fundamental HTML concepts including document structure, semantic elements, and basic page creation. I set up my development environment using Visual Studio Code and created my first simple webpage. The primary challenge was understanding the distinction between block-level and inline elements, which I resolved through practical exercises. I established a GitHub account in preparation for future version control requirements.

Reflection: I feel confident with basic HTML syntax, though I anticipate CSS will present a steeper learning curve.
week 2 — 13th january 2026
🎨 introduction to CSS styling
css selectors box model

This week focused on Cascading Style Sheets. I learned about selectors (element, class, ID), the box model (margin, padding, border, content), and basic colour theory for web design. I created a styled version of my initial HTML page, experimenting with different font families and background colours. The concept of specificity initially caused confusion, but after completing several exercises, I developed a clearer understanding of how CSS rules are prioritised.

Reflection: I enjoy the creative aspect of CSS. I need more practice with the box model, particularly margin collapsing behaviour.
week 3 — 20th january 2026
📐 layout techniques: flexbox and positioning
flexbox positioning layout

Layout techniques were the primary focus this week. I learned Flexbox for one-dimensional layouts and explored absolute, relative, fixed, and sticky positioning. These concepts were essential for creating the corkboard-style homepage I envisioned for my portfolio. Implementing absolute positioning for the polaroid images required precise measurement and significant trial and error. I also learned about z-index to manage stacking contexts.

Reflection: Flexbox is powerful but requires memorisation of numerous properties. Positioning remains challenging but is becoming more intuitive with practice.
week 4 — 27th january 2026
📱 responsive design and media queries
responsive media queries mobile-first

This week addressed the critical topic of responsive web design. I learned to implement media queries to adapt layouts across different screen sizes. For my portfolio, this meant redesigning the corkboard layout for mobile devices, converting the fixed-position polaroids into a vertical stacked format. The mobile-first approach was emphasised, though I retrospectively applied media queries to my existing desktop-oriented design. Testing across multiple device sizes revealed several edge cases requiring additional breakpoints.

Reflection: Responsive design is essential for accessibility. My mobile layout is functional but less aesthetically pleasing than the desktop version — an area for future improvement.
week 5 — 3rd february 2026
💻 version control with Git and GitHub
git github version control

Version control was introduced this week using Git and GitHub. I learned fundamental commands including init, add, commit, push, pull, and branch. I created a repository for my portfolio project and established a workflow of committing changes after each development session. Initially, I encountered merge conflicts when working across multiple devices, but I learned to resolve these using VS Code's built-in conflict resolution tools. Publishing my site via GitHub Pages was a significant milestone — seeing my work live online was highly motivating.

Reflection: Git is indispensable for project management. I now understand why version control is considered an industry standard practice.
week 6 — 10th february 2026
⚙️ JavaScript fundamentals
javascript variables functions DOM

JavaScript fundamentals were introduced this week. I learned about variables (let, const, var), data types, functions, conditionals, loops, and basic DOM manipulation. The Document Object Model concept was particularly important for understanding how JavaScript interacts with HTML. I created simple interactive features including a button that changes text colour and a basic image carousel. Asynchronous programming concepts were briefly introduced but will be covered in greater depth later.

Reflection: JavaScript is more complex than HTML and CSS. I find the syntax logical but need substantial practice to internalise core concepts.
week 7 — 17th february 2026
📋 form handling and validation
forms validation events

This week focused on HTML forms and client-side validation using JavaScript. I learned about form elements (input, textarea, select, button), event listeners (submit, input, change), and real-time validation techniques. I implemented a contact form with character counting, email format validation, and required field checking. Preventing default form submission behaviour and providing user feedback through DOM manipulation were key skills developed during these exercises.

Reflection: Form validation is more complex than anticipated, particularly providing clear, accessible error messages. I appreciate the importance of good UX in form design.
week 8 — 24th february 2026
🗄️ introduction to databases and Supabase
supabase postgresql database

Backend concepts were introduced through Supabase, a PostgreSQL-based backend-as-a-service platform. I learned about database tables, rows, columns, primary keys, and foreign keys. I created a contact_messages table with appropriate schema (id, name, email, message, created_at). Understanding row-level security (RLS) policies and API authentication methods required significant research. I successfully connected to my Supabase project using the provided API keys and tested basic read operations.

Reflection: Databases are conceptually straightforward but the surrounding infrastructure (APIs, security, authentication) is complex. Supabase simplifies many of these challenges.
week 9 — 3rd march 2026
🔗 integrating forms with Supabase
api integration fetch async/await

This week involved connecting my HTML contact form to the Supabase database using JavaScript's fetch API. I learned about HTTP methods (POST, GET), request headers, JSON formatting, and asynchronous programming using async/await syntax. Debugging CORS (Cross-Origin Resource Sharing) errors was challenging — I discovered that Supabase requires specific configuration to accept requests from external domains. After implementing the correct headers and testing extensively, I successfully stored form submissions in my database table.

Reflection: Asynchronous operations and API integration represent a significant conceptual leap. Seeing my form data appear in the Supabase dashboard was extremely rewarding after hours of debugging.
week 10 — 10th march 2026
🎨 portfolio design and CSS frameworks
design css frameworks custom css

This week was dedicated to refining my portfolio's visual design. I explored CSS frameworks including Bootstrap and Tailwind but ultimately chose to write custom CSS to achieve my specific "corkboard with polaroids" aesthetic. I established a consistent colour scheme using CSS variables, implemented custom scrollbar styling, and added subtle animations including a hover wiggle effect for interactive elements. I also created a cohesive visual language across all pages using shared class names and component-based styling.

Reflection: Custom CSS offers greater creative freedom than frameworks, though it requires more development time. I am satisfied with the distinctive, personal aesthetic I have achieved.
week 11 — 17th march 2026
🧪 usability testing and heuristic evaluation
usability nielsen heuristics

This week focused on evaluating my site against Nielsen's 10 Usability Heuristics. I identified three significant issues: (1) lack of consistent navigation between pages (violating "user control and freedom"), (2) insufficient error messaging for form validation (violating "error prevention"), and (3) poor mobile layout causing content overflow (violating "aesthetic and minimalist design"). I implemented fixes including a persistent home button, enhanced form feedback messages, and refined media query breakpoints. Peer testing provided additional insights.

Reflection: Usability testing revealed issues I had become blind to through familiarity. Heuristic evaluation is a valuable systematic approach to identifying UX problems.
week 12 — 24th march 2026
📝 content creation and page development
content copywriting multipage

This week involved creating substantive content for all portfolio pages. I wrote personal "About Me" copy, documented my interests, created project descriptions, and developed this 16-week devlog. I also implemented a sitemap page to improve site navigation and information architecture. Writing for the web required balancing personality with professionalism — I aimed for an authentic, slightly informal tone while maintaining academic appropriateness. I also optimised images for faster loading times.

Reflection: Content creation is as important as technical implementation. A visually appealing site with poor content fails to engage users effectively.
week 13 — 31st march 2026
♿ accessibility and web standards
accessibility wcag semantic html

Accessibility was the primary focus this week. I learned about WCAG (Web Content Accessibility Guidelines) principles including perceivable, operable, understandable, and robust. I implemented improvements including: semantic HTML structure (header, nav, main, article), alt text for all images, sufficient colour contrast ratios, keyboard navigation support, and ARIA labels where appropriate. I used browser developer tools and the WAVE accessibility evaluation tool to identify and resolve issues. These changes benefit all users, particularly those using assistive technologies.

Reflection: Accessibility should be integrated from the start rather than retrofitted. I will prioritise accessibility in future projects.
week 14 — 7th april 2026
⚡ performance optimisation
performance optimisation lighthouse

This week addressed website performance optimisation. I used Google Lighthouse to benchmark my site's performance, accessibility, SEO, and best practices. Initial scores revealed slow loading times primarily due to unoptimised images and external resources. I compressed all images, implemented lazy loading for below-the-fold content, minified CSS files, and moved JavaScript to the end of the body element. These changes improved my Lighthouse performance score from 67 to 89. I also added appropriate meta tags for SEO.

Reflection: Performance significantly impacts user experience and SEO. Small optimisations can yield substantial improvements in loading times.
week 15 — 14th april 2026
🔍 cross-browser testing and debugging
testing cross-browser debugging

Extensive cross-browser testing was conducted this week across Chrome, Firefox, Safari, and Edge. I discovered several inconsistencies: Flexbox behaviour varied slightly between browsers, custom scrollbar styling only worked in WebKit-based browsers, and some CSS grid properties required vendor prefixes. I resolved these issues using feature detection, graceful degradation, and standardised CSS properties where possible. I also tested on various physical devices including iPhone, iPad, and Android phones to verify responsive behaviour.

Reflection: Browser diversity remains a significant challenge in web development. Testing across multiple browsers is essential for professional-quality work.
week 16 — 21st april 2026
📦 final submission and project reflection
submission documentation portfolio

This final week was dedicated to preparing the complete portfolio for summative assessment. I compiled all required evidence including screenshots, the development lifecycle report, and the complete codebase. I verified that all pages function correctly, the contact form successfully stores data in Supabase, and the site meets all specified requirements. I also wrote comprehensive documentation within my code using comments to explain structural decisions and complex implementations. The GitHub repository was finalised with a clear README.

Reflection: Reflecting on 16 weeks of progress reveals substantial growth. I began with basic HTML and have developed a fully functional, database-connected portfolio. This module has provided a solid foundation for continued learning in web development.
